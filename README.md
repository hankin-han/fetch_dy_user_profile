# 抖音账号作品数据管理工具

**版本：v1.5**  
**适用环境：Chrome 浏览器控制台（F12 → Console）**  
**作者：hankin**  
**最后更新：2026-06-22**

---

## 功能简介

在抖音用户主页，通过 Chrome 开发者工具控制台注入此脚本，可获取该用户所有公开作品的结构化数据，并在页面侧边栏中以多种视图展示（表格/网格/仪表盘/对比/时间线），支持搜索、筛选、排序、预览，并支持导出为 JSON / CSV / TXT / 文案 TXT / 视频 ZIP 多种格式。面板支持全屏模式，打开时左侧有毛玻璃遮罩效果。

---

## 使用方法

1. 打开 Chrome 浏览器，访问抖音用户主页，例如：
   ```
   https://www.douyin.com/user/MS4wLjABAAAAjb1juHnK9tygA0nuoGgSEMW7ZuJzXNnTMx9XwaQh19k
   ```
2. 按 `F12` 打开开发者工具，切换到 **Console（控制台）** 标签
3. 将 `fetch_dy_user_profile.js` 的完整代码复制粘贴到控制台，按回车执行
4. 页面右侧会出现一个侧边抽屉面板，所有功能均在面板内操作

> ⚠️ 必须使用**登录状态下的抖音账号**访问，否则无法获取完整数据。

## 功能详细说明

### 1. 用户信息展示

面板顶部展示当前用户的核心信息，带有背景封面图：

- **背景封面图**：取 `cover_and_head_image_info.profile_cover_list` 中的封面图，作为用户信息卡片的 CSS 背景图
  - 浅色模式遮罩色取 `light_cover_color`，深色模式取 `dark_cover_color`（hex → rgba，alpha 0.82/0.85）
  - 通过 `::before` 伪元素做半透明遮罩，确保文字始终清晰可读
  - 无封面图时使用默认遮罩色（浅色白色 0.82，深色深灰 0.85）
- **头像**：自动抓取用户真实头像（`avatar_thumb` / `avatar_medium`），无头像时降级显示昵称首字
- **下载头像**：头像下方按钮，点击通过 `fetch` 下载最高清头像（`avatar_larger` → `avatar_medium` → `avatar_thumb`）
- **下载背景**：json原数据按钮旁，点击下载背景封面原图
- 昵称、抖音号（unique_id）
- 签名、IP 属地
- 粉丝数、关注数、作品数、获赞总数
- 作品数在数据拉取完成后会自动更新为实际加载数（与底部列表总数一致）

### 2. 毛玻璃遮罩

面板打开时，页面左侧会出现毛玻璃遮罩效果（`backdrop-filter: blur(8px)` + 半透明黑色），类似弹窗遮罩。点击遮罩区域可关闭面板。

### 3. 全屏模式

面板右上角关闭按钮旁提供**全屏切换按钮**（四箭头展开图标）：

- **进入全屏**：面板铺满整个视口，毛玻璃遮罩自动隐藏
- **退出全屏**：恢复原始 1180px 右侧面板尺寸
- **全屏表格优化**：标题列和标签列按百分比分配宽度（30% + 20%），自动换行完整显示，其余列压缩到最小宽度
- 关闭面板、点击遮罩均会自动退出全屏状态

### 4. 主题切换

面板右上角提供 **浅色 / 深色** 主题切换按钮，切换后自动记忆偏好（localStorage），下次打开时自动应用。

### 5. 视图切换

支持五种视图模式，通过工具栏视图切换下拉菜单选择：

| 视图 | 类型 | 说明 |
|------|------|------|
| **表格视图** | 浏览 | 传统行列表格，适合查看详细数据和排序 |
| **网格视图** | 浏览 | 卡片式封面网格，适合快速浏览作品封面 |
| **仪表盘** | 分析 | 聚合统计：总览卡片、类型分布、Top 10 排行榜、互动均值 |
| **对比视图** | 分析 | 选中 2-6 条作品并排对比各项指标，高亮每项最高值 |
| **时间线** | 分析 | 按月分组展示发布节奏，每月统计作品数和平局点赞 |

视图偏好自动记忆，下次打开时恢复。

### 6. 搜索过滤

工具栏提供搜索框，支持按 **作品 ID**、**标题关键词** 或 **话题标签** 过滤，输入后点击按钮或按回车执行搜索，点击「重置」清除搜索条件。

- **标签点击搜索**：表格视图下，标签列中的每个话题标签可点击，点击后自动填入搜索框并执行搜索，快速筛选同标签作品

### 7. 列选项

点击工具栏「列选项」按钮，可自定义表格中显示的列，勾选/取消勾选即可实时生效。

默认显示的列：序号、封面、标题、时间、点赞、分享、评论、收藏、推广、操作  
默认隐藏的列：**类型、作者、标签**（可在列选项中勾选显示）

**标签列**：从 `text_extra[].hashtag_name` 提取话题标签，以蓝色 `#话题名` 小标签形式展示，点击可搜索。

### 8. 排序

表格视图下，点击表头含排序箭头的列（时间、点赞、分享、评论、收藏、推广），即可按该字段排序，再次点击切换升序/降序。

### 9. 封面预览

- **视频作品**：点击封面图，弹出全屏视频播放弹窗，自动播放
- **图文作品**：点击封面图，弹出全屏图片幻灯片，支持：
  - 左右箭头切换图片
  - 底部圆点指示器，点击跳转
  - 左上角显示 `当前图 / 总图数`
  - 自动播放该作品关联的背景音乐
  - 键盘 ← → 切换，ESC 关闭

### 10. 分页

表格、网格、时间线视图均支持分页，底部分页控件提供每页条数选择（20 / 50 / 100 / 200 / 500），选择自动记忆。

### 11. 全选 / 取消全选

工具栏提供「全选」「取消全选」按钮，方便批量操作。

### 12. 数据分析视图

#### 仪表盘 📊

从全局视角聚合分析全部作品数据，无需选择（基于 Chart.js 渲染）：

- **概览卡片**：作品总数、累计点赞、累计评论、累计分享、累计收藏（5 项核心指标）
- **类型分布**：环形图（Doughnut）展示视频 vs 图文占比，悬停显示具体数量和百分比
- **互动均值**：柱状图（Bar）对比点赞/评论/分享/收藏四项平均值，四色区分
- **点赞 Top 10**：水平柱状图（Horizontal Bar），前 3 名深红高亮，悬停显示具体数值
- **评论 Top 10**：水平柱状图，前 3 名深蓝高亮，悬停显示具体数值
- **分享 Top 10**：水平柱状图，前 3 名琥珀色高亮，悬停显示具体数值
- **收藏 Top 10**：水平柱状图，前 3 名绿色高亮，悬停显示具体数值
- **布局**：4 个 Top 10 图表两行两列排列，标题截断优化（最多 25 字符 + 省略号）

- **图表自适应**：浅色/暗色模式自动适配坐标轴和文字颜色

#### 对比视图 ⚖️

将选中的作品并排对比，直观看出哪条作品在哪项指标上胜出：

- **使用方式**：先在表格或网格视图中勾选 2-6 条作品，再切换到对比视图
- **封面卡片**：顶部并排展示每条作品的封面和标题
- **指标表格**：发布时间、类型、点赞、评论、分享、收藏六项指标横向对比
- **最高值高亮**：每项指标的最高值标红并加 👑 标识

#### 时间线视图 🕐

按时间维度梳理作品发布规律：

- **按月分组**：自动将当月页数据按月份归组
- **月度统计**：每月显示作品数量和平均点赞
- **时间轴节点**：每条作品以竖线时间轴 + 封面缩略图 + 标题 + 四项互动数据展示
- **数据源**：当前搜索/筛选结果（`filterWorks`），支持分页翻看

### 13. 导出数据

点击工具栏「导出数据 (N) ▾」，下拉菜单分为三个区域：

#### 导出选择（操作当前勾选的作品）

| 格式 | 说明 |
|------|------|
| **JSON** | 结构化数据，含完整用户信息、导出时间、作品列表（含视频地址、播放地址、各项统计数据） |
| **CSV** | 逗号分隔表格，UTF-8 BOM 编码，Excel / WPS 直接打开不乱码 |
| **TXT** | 纯文本格式，用户信息 + 每条作品详细数据，适合阅读 |
| **仅文案TXT** | 仅导出选中作品的文案内容，每条带序号编号（如 `1. 文案内容`），空行分隔 |

#### 批量导出全部（操作全部作品，无需勾选）

| 格式 | 说明 |
|------|------|
| **全部JSON** | 全部作品导出为 JSON 格式 |
| **全部TXT** | 全部作品导出为 TXT 格式 |
| **全部CSV** | 全部作品导出为 CSV 格式 |
| **全部文案TXT** | 全部作品文案导出为 TXT，每条带序号编号，空行分隔 |

#### 导出视频

| 格式 | 说明 |
|------|------|
| **导出视频 (.zip)** | 批量下载选中视频的原文件，打包为 ZIP 压缩包 |

导出文件命名格式：`抖音作品_昵称_选中数据.后缀`（全部导出为 `抖音作品_昵称_全部数据.后缀`，视频 ZIP 为 `抖音视频_昵称_日期.zip`，文案 TXT 为 `抖音文案_昵称_选中.txt` / `抖音文案_昵称_全部.txt`）

#### 文案提取说明

文案导出支持增强提取，能从多个字段兜底获取作品的文字内容：

- `desc`（作品标题/描述）
- 话题标签（`#话题` 格式）
- 歌词、评论置顶等文字内容（如有）
- 文本已做 HTML 转义处理，避免特殊字符（`<`、`>`、`&`、`"`）导致 UI 或文件内容异常
- 每条文案前带序号编号（如 `1. 今天天气真好`、`2. 周末愉快`），无文案时显示 `[作品N：无文案]`

#### 导出内容说明

**JSON 格式**：
```json
{
  "user_info": {
    "nickname": "昵称",
    "unique_id": "抖音号",
    "sec_uid": "...",
    "signature": "签名",
    "avatar": "头像链接",
    "ip_location": "IP属地",
    "region": "地区",
    "follower_count": 粉丝数,
    "following_count": 关注数,
    "aweme_count": 作品数,
    "total_favorited": 获赞总数,
    "page_url": "当前页面链接"
  },
  "export_time": "导出时间",
  "select_total": 选中数,
  "list": [ ... 作品详情 ... ]
}
```

**TXT 格式**：文件头部包含完整用户信息和页面链接，每条作品以分隔线隔开，键值对缩进对齐。

#### 视频导出（ZIP）

选中视频作品后，选择「导出视频 (.zip)」，脚本会自动下载所有选中视频并打包为一个 ZIP 文件。

- **文件命名**：ZIP 内每个视频为 `作品ID_标题.mp4`
- **ZIP 命名**：`抖音视频_昵称_2026-06-21.zip`
- **仅限视频**：图文类型作品自动跳过，仅导出视频
- **压缩策略**：`level:0` 不压缩（视频本身已压缩，再压缩无意义且浪费时间）
- **进度展示**：全屏进度浮层，支持中途取消
- **下载失败**：单个视频失败不影响其他视频，最终弹窗汇总成功/失败数

**下载策略（四级降级）**：

抖音视频地址存在 CORS 限制和重定向问题，脚本采用四级降级策略确保最大化下载成功率：

```
① HEAD play_addr (同源) → 读到 Location 头 → 拿到真实 CDN 地址
② 匿名 fetch CDN 地址 (credentials: omit, 不带 cookie) → CDN 通常允许匿名 CORS ✅
③ 匿名还是 opaque → 再试带 cookie 的 fetch
④ 仍失败 → 回退新标签页
```

| 步骤 | 操作 | 说明 |
|------|------|------|
| ① HEAD 重定向 | `fetch(url, { method: "HEAD", redirect: "manual" })` | 抖音 play_addr 是同源重定向地址，HEAD 请求读取 301/302 的 Location 头拿到真实 CDN URL |
| ② 匿名 fetch | `fetch(cdnUrl, { credentials: "omit" })` | 不带 cookie 请求 CDN，CDN 通常允许匿名 CORS，避免 cookie 导致的拒绝 |
| ③ 带 cookie fetch | `fetch(cdnUrl, { credentials: "include" })` | 匿名失败时尝试带 cookie 再试 |
| ④ 多候选地址 | play_addr → bit_rate → download_addr | 依次尝试所有候选地址，全部失败才跳过该视频 |

**打包流程**：

```
阶段 1（0% → 75%）：逐个视频调用 fetchVideoBlob() 收集到内存
阶段 2（75% → 100%）：fflate.zipSync() 打包为 ZIP → 触发下载
```

**依赖库**：

- **fflate**（`v0.8.2`）：流式 ZIP 打包库，通过动态 `<script>` 标签加载 UMD 版本，无需构建工具
  ```html
  <script src="https://cdn.jsdelivr.net/npm/fflate@0.8.2/umd/index.js"></script>
  ```

## 技术说明

### 第三方依赖

| 库 | 版本 | 用途 | 加载方式 |
|-----|------|------|---------|
| [Tailwind CSS](https://tailwindcss.com) | CDN | UI 样式框架 | 动态 `<script>` 标签 |
| [fflate](https://github.com/101arrowz/fflate) | 0.8.2 | ZIP 打包（视频导出） | 动态 `<script>` 标签（UMD） |
| [Chart.js](https://www.chartjs.org) | 4.4.7 | 统计图表（仪表盘） | 动态 `<script>` 标签（UMD） |

### 数据来源

脚本通过拦截/调用抖音 Web 端 API 获取数据：

| API 接口 | 用途 |
|-----------|------|
| `/aweme/v1/web/user/profile/other/` | 获取用户基本信息 |
| `/aweme/v1/web/aweme/post` | 分页获取用户作品列表 |

### sec_uid 动态提取

脚本自动从当前页面 URL 中动态提取 `sec_uid`，无需手动配置，兼容各种格式的抖音用户链接。

### 翻页策略

- 每次请求 `count=35`（减少请求次数）
- 根据 `has_more` 判断是否继续翻页
- 空页（接口抖动）不立即退出，允许最多连续 3 页为空才停止
- `max_cursor` 不变时自动停止，防止死循环
- 拉取过程中显示进度条

### 图片幻灯片

图文作品的多张图片以幻灯片形式展示，同时自动播放该作品关联的音乐（取 `item.music.play_url.url_list[0]`）。

### 背景封面图实现

用户信息卡片的背景封面图采用 CSS `background-image` 实现，而非 `<img>` 标签：

- **数据来源**：`userProfile.cover_and_head_image_info.profile_cover_list[0].cover_url.url_list[0]`
- **遮罩色**：浅色模式取 `light_cover_color`，深色模式取 `dark_cover_color`（hex → rgba，alpha 0.82/0.85）
- **实现方式**：通过 CSS 变量 `--cover-bg` / `--cover-overlay` / `--cover-overlay-dark` 动态设置，`::before` 伪元素做半透明遮罩，内容 `z-index:1` 确保文字可读

### 全屏模式实现

全屏模式通过 CSS class `fullscreen` 控制，与 `open` 状态独立管理：

- **进入全屏**：`wrap.classList.add("fullscreen")`，CSS 强制 `right:0; left:0; top:0; bottom:0; width:100%; border-radius:0`
- **遮罩隐藏**：全屏时 `#dy-drawer-mask` 的 `opacity:0; pointer-events:none`
- **表格列宽**：全屏时 `table-layout:auto`，标题列 `width:30%`、标签列 `width:20%`，其余列固定像素宽度，`!important` 覆盖 inline style
- **退出同步**：关闭面板、点击遮罩、切换 open 状态时均自动移除 `fullscreen` class

## 注意事项

1. **登录状态**：必须在登录状态下使用，否则 API 返回数据不完整
2. **私密/已删除作品**：`aweme_count` 包含私密和已删除作品，实际能拉取的数量可能少于显示的总数，属于正常现象
3. **频率限制**：抖音有接口频率限制，脚本已设置每次请求间隔 800ms，如遇拉取中断，可重新执行脚本
4. **仅限个人使用**：本脚本仅供个人数据备份和学习研究使用，请勿用于商业目的或违规行为
5. **IP 属地字段**：抖音 API 返回字段名可能变化，脚本已做多字段兜底（`ip_location` / `ip_attr` / `region` / `address` / `city`）
