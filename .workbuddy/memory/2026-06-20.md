## 视频导出功能迭代

经历了多轮方案变更：
1. 最初：fflate + fetch（zip 打包）→ CORS 失败
2. 改成 Tampermonkey + GM_xmlhttpRequest → 用户不要 Tampermonkey
3. 改成 <a download> 触发浏览器下载 → 跨域 CDN 忽略 download 属性，变成新标签页播放
4. 最终：fetch 同源 play_addr URL (redirect: follow) → 拿 blob → blob URL → <a download> 下载；CDN 无 CORS 头时 fallback 到新标签页打开 + 提示用户右键保存

`exportVideoZip()` 核心逻辑：
- 候选地址优先级：play_addr > bit_rate > download_addr
- 依次 fetch 尝试，response.type === "opaque" 则跳过当前地址
- 成功拿到 blob → URL.createObjectURL → <a download> 触发下载
- 全部地址都失败 → window.open 回退到新标签页
- 结束后统计 successCount / fallbackCount / failCount 并弹提示

## 视频导出：匿名 CORS 策略优化

CDN 全部返回 opaque response，带 cookie 的请求被 CORS 拒绝。改为两阶段下载：
1. HEAD 同源 play_addr（redirect: manual）拿真实 CDN 地址
2. 匿名 fetch CDN 地址（credentials: omit）→ 很多 CDN 支持匿名 CORS
3. 匿名失败再尝试带 cookie 的 fetch
4. 仍失败才回退新标签页

新增 `triggerDownload(blob, filename)` 辅助函数封装 blob URL 下载逻辑。

## 2026-06-21 视频导出：加入 zip 打包

用户确认匿名 CORS 方案可以下载视频，改为 zip 打包模式：
- 阶段 1：逐个 fetchVideoBlob()，收集所有 blob 到内存，进度条 0-75%
- 阶段 2：fflate.zipSync() 一次性打包所有 mp4，进度条 75-100%
- 触发 zip 文件下载（文件名：抖音视频_{nickname}_{date}.zip）

新增函数：waitFflate()（等 CDN 加载）、fetchVideoBlob()（抽离下载逻辑）
fflate CDN：https://cdn.jsdelivr.net/npm/fflate@0.8.2/umd/index.js（需用 UMD 格式）
注：zipSync 用 level:0 不压缩，视频文件已经压缩了
