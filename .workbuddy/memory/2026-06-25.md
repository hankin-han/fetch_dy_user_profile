## 修复腾讯云 ASR URL 模式下载失败

**问题**：文件 >5MB 时 `SourceType=0` URL 模式，`$baseUrl` 写成 `https://`，服务器未配 SSL。改成 `http://` 后仍失败 — 根因是腾讯云 ASR 服务网络无法访问该 CVM 的 80 端口（安全组问题）。

**最终方案**：前端自动压缩。添加 lamejs 纯 JS MP3 编码器，>5MB 文件用 Web Audio API 解码 → 16kHz 单声道 → 32kbps MP3，压缩后 <5MB 则 base64 直传。不可用时降级到 16kbps/8kbps。

## 前端音频压缩实现（txt.html）

- 引入 `lamejs@1.2.0` CDN（纯 JS MP3 编码器）
- `compressAudio(file, targetKbps)`: 异步压缩，自动降码率重试（32→16→8kbps）
- `doCompress(file, kbps)`: 核心：FileReader → decodeAudioData → 混音单声道 → 降采样 16kHz → lamejs 编码
- `resampleLinear()`: 线性插值降采样
- `submitFile()`: 优先用 `compressedFile`（Blob），否则用 `selectedFile`
- 播放器仍用原始文件（音质好），ASR 用压缩文件
- 显示压缩前后大小对比（e.g. "9.00 MB → 2.50 MB (已压缩 72%)"）

## 腾讯云 ASR Demo

新增腾讯云录音文件识别方案，比百度方案更适合长音频（无分段限制）。

### 文件
- `tencent_submit.php` — 收文件 → TC3-HMAC-SHA256 签名 → CreateRecTask
- `tencent_poll.php` — DescribeTaskStatus 轮询，返回结果并缓存
- `txt.html` — 前端上传 + 每 3 秒轮询进度

### 架构
- ≤5MB：base64 直传（SourceType=1），不依赖公网 URL
- >5MB：URL 方式（SourceType=0），需服务器 HTTPS 公网可访问
- 引擎：16k_zh，ResTextFormat=3（字幕场景，按标点分段）
- 支付：每月 10 小时免费额度，超出 ¥0.001/秒
- 缓存：`cache/tx_{md5}.json`
- 日志：`logs/tencent_YYYY-MM-DD.log`

### 部署
- 改 `tencent_submit.php` 和 `tencent_poll.php` 中的 SecretId/SecretKey
- `$baseUrl` 需改为服务器的 HTTPS 地址
- 确保 `uploads/`、`cache/`、`logs/` 有写权限
